## DeOpenRouter

**Transparent and Trust-Minimized AI API Marketplace**

---

## Project Description

近年来，大量 AI 模型通过 API 形式被封装并转售，在实际使用中逐渐形成了所谓的“API 中转市场”。然而，这类市场普遍存在严重的不透明问题：用户无法验证所调用的模型是否真实、API 定价机制不公开、调用次数和费用缺乏可审计性。这种高度依赖中介信任的模式，使得用户和服务提供者都暴露在信息不对称和潜在欺诈风险之下。

本项目旨在设计并实现一个基于区块链的去中心化 AI API 市场协议，通过引入智能合约与链上可验证状态，减少对中心化中介的信任依赖。系统的核心目标是实现三个方面的透明性：价格透明（transparent pricing）、调用透明（auditable usage）、以及参与者身份透明（verifiable providers）。

在系统设计上，区块链主要承担“信任协调层”的角色。具体而言，所有 API 提供者需要在链上注册其服务，包括模型声明、接口标识以及调用价格。用户在调用 API 时，通过智能合约完成支付与调用记录的绑定，从而保证每一次调用行为都具有不可篡改的链上记录。同时，引入抵押与惩罚机制（staking and slashing），要求服务提供者质押一定资金，以提高作恶成本并增强系统的经济安全性。

考虑到区块链在计算效率和成本方面的限制，实际的模型推理过程仍然在链下执行。系统通过链上记录请求与响应的哈希值，实现调用行为的可审计性。在进一步扩展中，本项目还将探索结合 Zero-Knowledge Proof 或可验证计算技术，以增强对推理过程正确性的验证能力，尽管该部分在当前实现中可能采用简化或模拟方案。

最终，本项目将实现一个端到端的原型系统，包括：链上智能合约（用于 API 注册、支付结算与日志记录）、链下推理服务，以及一个简单的前端界面用于 API 浏览与调用。通过该系统，用户可以直接选择不同的 API 提供者，比较公开价格，并在链上验证其调用历史，从而显著提升整个 API 市场的透明性与可信度。

---

## Expected Contributions

本项目的主要贡献包括：

首先，从系统设计角度提出一种“链上信任 + 链下计算”的混合架构，将区块链作为信任基础设施，而非计算平台。其次，构建一个透明的 API 市场机制，使价格、调用记录和资金流动均可公开验证。最后，通过引入经济激励与惩罚机制，探索在缺乏完全密码学验证的情况下，如何通过激励设计实现系统的安全性与可靠性。
