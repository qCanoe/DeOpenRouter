~0:45 · Opening

Story

The problem: centralized AI gateways are trust-heavy

Typical routers consolidate pricing, routing, billing, and disputes under one operator. That is convenient, but users cannot easily verify quiet rule changes, whether a charge matched the published policy, or whether records could be rewritten after the fact.

  • Our narrative hook: which parts of the workflow belong on an EVM so rules and settlement are publicly observable and harder to tamper with?
  • We do not claim fully on-chain inference; we trade realism for clearer settlement and auditability.

~1:35 · Story

Story · Goals

What we move on-chain (README)

  • Provider identity commitments and metadata registered publicly
  • Price updates are not instantly effective; they have observable delay
  • Each paid call leaves an auditable on-chain record
  • Stake can be reduced via an on-chain slashing proposal with a challenge window; proceeds go to configurable slashTreasury
  • External audit outcomes can be anchored as report hashes

Inference stays off-chain for practicality; settlement and auditing become more transparent.

Story · Scope

MVP prototype, not a production inference network

The README states this repository is an MVP focused on the trust and settlement model, not a production-grade decentralized inference network. Stating limits up front strengthens the story.

On-chain

Coordination, pricing, payments, audit anchoring, slashing hooks, and governance parameters.

Off-chain

Real inference; relay in mock mode or proxied to OpenRouter; audit probes and canonical JSON handling.

~0:50 · Implementation

Flow

End-to-end journey (five steps)

  1. Provider registers on-chain: price, stake, metadata URI, identity and capability hashes, endpointCommitment
  2. Web lists providers; users can simulate calls via the relay API
  3. Client hashes request and response, then calls on-chain invoke(...) with payment
  4. Contract transfers fees, stores the call record, emits CallRecorded
  5. If auditing is enabled: probe relay off-chain → canonical JSON → optional off-chain publish → recordAudit / recordAuditWithUri anchors keccak256(canonicalJson)

Implementation · Solidity

DeOpenRouterMarketplace.sol

Primary contract under contracts/src/. Function names match the README for direct mapping to the codebase.

~0:55 · Implementation

Story · Assumptions

README warning: economic security

Economic safety still depends on multisig and treasury configuration. The chain implements slashing delay + challenge and auditor allowlisted multisig, but there is no full arbitration market, escrow settlement, or cryptographic proof of correct inference.

See docs/AUDIT_GOVERNANCE.md and docs/TRUST_LAYERS.md.

~0:35 · Story

Story · Boundaries

What the MVP does not claim (README)

  • On-chain endpointId commitment only; raw endpoint URLs are not published on-chain
  • Call records store request/response hashes, not full plaintext
  • Instant per-call settlement; no escrow, streaming, or async settlement
  • Slash proposals originate from slashOperator with a challenge period; multi-auditor quorum aggregation is largely off-chain
  • No decentralized discovery, reputation, automated arbitration market, or inference execution proof network
